# **App Name**: IRTRA Explorer

## Core Features:

- Location Search & Filtering: Allow users to search for places within the complex and filter results by categories such as restaurants, hostals, or events, providing a dynamic list of relevant locations.
- Categorized Exploration: Enable users to visually browse and explore different categories of attractions (restaurants, hostals, spas, pools, events, information), each presented with modern, attractive elements and clear iconography.
- Detailed Place View: Present comprehensive information for each location, including a main image, name, category, location, operating hours, a brief description, and action buttons (e.g., 'Ver ubicación en mapa', 'Cómo llegar').
- Home Screen Overview: Feature a main home screen with an inviting hero header, a prominent search bar, sections for visual categories, highlighted featured places, and a clear visual placeholder for the complex map.
- Visual Map Placeholder: Integrate a visually attractive, non-functional placeholder for a future map, showing illustrative markers and a tourist map style on both the home screen and place detail screens.
- Hardcoded Content Display: Utilize hardcoded data for all locations, categories, and descriptive content, ensuring full functionality of search, detail views, and listings without a backend connection for the MVP.

## Style Guidelines:

- Primary color: Vibrant blue (#1F6BFF) for main navigation, interactive elements, and prominent headers, conveying modernity and clarity.
- Accent colors: Orange (#FF8C42) and Yellow (#FFD166) to highlight secondary actions and visually interesting elements, adding warmth and friendliness.
- Backgrounds: A base light blue-gray (#F4FAFD) for overall screens, a soft warm tone (#FFF4EC) for complementary sections, and crisp white (#FFFFFF) for card surfaces and inputs, with a linear-gradient for hero sections (from #8ED6F7 to #F4FAFD).
- Text colors: Dark headings (#1E293B) for strong hierarchy, moderate body text (#475569) for readability, and muted tones (#94A3B8) for secondary information or disabled states.
- Headline fonts: 'Poppins' (sans-serif, geometric) for headings (h1, h2) to achieve a modern, rounded, and inviting feel. This font choice aligns with the 'rounded modern' requirement.
- Body fonts: 'Inter' (sans-serif, grotesque) for body text and captions (body, caption) ensuring excellent readability and a neutral, objective contrast with the headlines, crucial for mobile-first design.
- Utilize modern, clean iconography for categories, navigation, and actions, ensuring visual consistency and intuitive understanding across the application. Icons should support the 'touristy, friendly' vibe.
- Consistent border-radius application: 24px for containers, 18px for buttons and cards, and 16px for images, providing a soft, friendly, and cohesive visual language throughout the mobile interface.
- Soft elevation shadows: Apply '0px 6px 18px rgba(0, 0, 0, 0.08)' to key elements like cards and interactive components to provide subtle depth and emphasize hierarchy, avoiding flat or corporate aesthetics.
- Mobile-first grid system: Implement a consistent layout grid with 20px margins and 16px spacing between elements, ensuring content is always centered, proportional, and optimized for mobile viewports without text overflow.
- Floating rounded bottom navigation bar: A visually distinctive bottom navigation with an active state in the primary blue (#1F6BFF) and an inactive state in muted grey (#64748B) to guide user navigation effectively.
- Subtle and functional animations: Incorporate smooth transitions for screen changes, card interactions, and button presses to enhance the user experience and convey responsiveness, without distracting from content.
